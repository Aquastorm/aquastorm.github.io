Project Description

Main Tasks

  • Definition of the project architecture and assets organization.
  • Built many core functionalities, e.g., code used by multiple systems to locally store/read game data to/from the device's local storage, network code to access web services.
  • Developed the following systems:
    • Localization.
    • Scenes management and navigation.
    • Screens and popups management and navigation.
    • The client-side code of a remote configuration system, to remotely drive many aspects of the game (e.g., amount of points required to unlock the cars the player can drive, amount of points delivered by the questions of the in-game quiz).
  • Game profiling and use of various optimizations techniques to improve game performance.
  • Programmed the client-side of an in-house analytics system to send analytics events to an analytics back-end server.
  • Implemented the functionality to generate and scan QR codes to add other players to the actual list of friends.
    • {% comment %} • Achievement system to manage and track user actions and the corresponding number of points (stars) assigned to unlock parts of the game. E.g.: the first time the player finishes a race, she receives a defined number of stars. With enough stars won with similar achievements, the player can unlock a new car. • Developed the system to calculate the optimal path the opponent cars’ AI should follow on the tracks that are dynamically generated based on the track physically build by the player. • Dynamic generation of the virtual racetracks (environments, road, colliders, spawn points, mini map, etc.) based on the track data generated when building the physical racetrack using the hexagonal tiles in the AR module. • Development of a track builder tool for internal purposes. This enabled Staay employees to test many aspects of the race module and dynamic track generation without using the AR module (which recognizes the physical track tiles and generates data used to build the virtual track). • Development of the main module’s UI related code. {% endcomment %}
